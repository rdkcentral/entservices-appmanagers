SCENARIO 4: Deferred WindowManager Telemetry (Optional Enhancement)
================================================================================

Current Implementation (Eager):
───────────────────────────────
  RDKWindowManager.Initialize()
    └─ RDKWindowManagerTelemetryReporting::getInstance().initialize(service)
       └─ Immediate attempt to connect to TelemetryMetrics
       └─ If fails: Warning logged, continues


Alternative: Lazy Implementation (Not required, but possible):
──────────────────────────────────────────────────────────────
  RDKWindowManager.Initialize()
    └─ Skip telemetry init (defer to first use)
    └─ Continue normally ✅

  When recordCreateDisplayTelemetry() called:
    ├─ Check if telemetry enabled (compile flag)
    ├─ Check if available (cached)
    ├─ If not: Initialize now
    │  └─ ensureTelemetryClient()
    │     ├─ Lock mutex
    │     ├─ QueryInterfaceByCallsign if needed
    │     └─ Cache
    ├─ Record telemetry ✅
    └─ Future calls use cache


Benefits of Lazy Approach:
──────────────────────────
  ✅ Faster WindowManager startup (skip one QueryInterfaceByCallsign)
  ✅ Works if TelemetryMetrics not yet active
  ✅ Same performance after first call (cached)
  ✅ Non-blocking either way

Risks of Lazy Approach:
──────────────────────
  ✅ None identified - already used by other plugins
  ✅ Thread-safe (uses mutex)
  ✅ Graceful degradation if TelemetryMetrics unavailable

WINDOWMANAGER SHELL THREAD ANALYSIS
================================================================================

Thread Lifecycle:
─────────────────

  Initialize():
    ├─ mEventListener = std::make_shared<Listener>()
    ├─ CompositorController::setEventListener(mEventListener)
    ├─ enableInactivityReporting(true)
    │
    ├─ shellThread = std::thread([=]() {
    │     std::lock_guard<std::mutex> lock(gRdkWindowManagerMutex)
    │     RdkWindowManager::initialize()
    │     PluginHost::ISubSystem* subSystems = service->SubSystems()
    │     // ... setup ...
    │     gRdkWindowManagerMutex.unlock()  // Release so main can continue
    │
    │     while (sRunning) {
    │       const double maxSleepTime = (1000 / gCurrentFramerate) * 1000
    │       double startFrameTime = RdkWindowManager::microseconds()
    │
    │       gRdkWindowManagerMutex.lock()
    │       
    │       // Process display creation requests
    │       while (gCreateDisplayRequests.size() > 0) { ... }
    │
    │       // Take screenshot if needed
    │       if (gNeedsScreenshot) { ... }
    │
    │       // Render frame
    │       RdkWindowManager::draw()
    │       RdkWindowManager::update()
    │
    │       gRdkWindowManagerMutex.unlock()
    │
    │       // Sleep for frame rate regulation
    │       double frameTime = RdkWindowManager::microseconds() - startFrameTime
    │       if (frameTime < maxSleepTime) {
    │         usleep(maxSleepTime - frameTime)
    │       }
    │     }
    │   })
    │
    ├─ RDKWindowManagerTelemetryReporting::getInstance().initialize()
    │  └─ Non-blocking telemetry init
    │
    └─ Return ✅

  Deinitialize():
    ├─ sRunning = false
    ├─ shellThread.join()  // Wait for thread to exit loop
    ├─ RdkWindowManager::deinitialize()
    └─ mEventListener.reset()

  ├─ Initialize: ~1-10ms (non-blocking)
  ├─ Shell thread: Runs independently
  ├─ Rendering loop: 60fps framework (e.g., 16.67ms per frame)
  ├─ Display creation: Queued and processed in loop cycles
  └─ Telemetry: Async, non-blocking ✅


No Blocking Concerns:
